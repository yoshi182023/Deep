Why Not Standard Python Scripts?
为什么不用标准的Python脚本？

Much of the work done in ML is done in this environment called a "notebook", a document format specialized for data processing and ML use cases, and specified in the iPython Notebook standard.
机器学习中大部分工作都在称为"笔记本"的环境中进行，这是一种专门用于数据处理和ML用例的文档格式，在iPython Notebook标准中指定。

Notebooks are a different paradigm than standard Python files with inline comments.
笔记本与带有内联注释的标准Python文件的范式不同。

They read and behave more like a field journal for a data scientist.
它们读起来和行为更像数据科学家的现场日志。

In addition to code, they include commentary the authors made to themselves and/or their readers while writing and executing the code.
除了代码之外，它们还包括作者在编写和执行代码时对自己和/或读者所做的注释。

They can also include a snapshot of the output generated by their code blocks, including images generated by graphing and plotting modules.
它们还可以包括由其代码块生成的输出的快照，包括由图形和绘图模块生成的图像。

Thus, notebooks can provide a much richer picture of how a result got computed.
因此，笔记本可以提供关于如何计算结果的更丰富的图像。

Unlike a traditional, deterministic software project, ML projects produce useful results by statistically modeling desired output.
与传统的确定性软件项目不同，ML项目通过统计建模所需输出来产生有用的结果。

Having a record of the output for each step in the process is crucial for improving or reconstructing results.
记录流程中每一步的输出对于改进或重建结果至关重要。

So, the journal-like features of a notebook are essential for big data and ML projects.
因此，笔记本的日志式功能对于大数据和ML项目是必不可少的。

There are a few broad resource types to pay attention to:
需要注意的资源类型有以下几种：

Processing Units (CPU/GPU/TPU): Where all the math required to transform the data and produce outputs is parallelized and executed.
处理单元（CPU/GPU/TPU）：所有转换数据和生成输出所需的数学运算都在此处进行并行化和执行。

GPUs/TPUs are purpose-built to handle higher-precision floating-point numbers and perform computations on thousands of numbers at a time using matrix math.
GPU/TPU是专门为处理更高精度的浮点数而构建的，可以使用矩阵数学同时对数千个数字执行计算。

With practice, you'll find you're learning and coding much faster with LLMs than without them.
通过练习，你会发现使用LLM比不使用LLM学习和编码速度快得多。

WARNINGS:
警告：

While LLMs create helpful rough draft code, don't trust it until you've fully audited and understood it!
虽然LLM创建有帮助的粗略草稿代码，但在你完全审计和理解它之前不要信任它！

LLMs frequently bury subtle or overt errors in confidently worded responses, and your peers will ultimately hold you accountable for any code you commit.
LLM经常在措辞自信的回应中掩埋微妙或明显的错误，你的同行最终会对你提交的任何代码负责。

Current best practice is treating LLM-generated code as a good hint, like the third most popular Stack Overflow answer.
目前的最佳实践是将LLM生成的代码视为一个好的提示，就像第三最受欢迎的Stack Overflow答案一样。

AI autocomplete and equivalent features within IDEs are particularly problematic because they eagerly generate code and text without explicit prompting; either giving away the answer to a question that is useful to reason about, or creating rabbit holes of unproductive tangents.
IDE中的AI自动完成和等效功能特别成问题，因为它们急切地生成代码和文本而没有明确的提示；要么给出一个有用的推理问题的答案，要么创建无生产力的切线兔子洞。

As such, labs and project notebooks will indicate when it is recommended to use LLM tools for code generation.

Review: Ways to Protect the User Environment from Pollution
审查：保护用户环境不被污染的方法

The best technique to avoid modifying the user shell environment from your notebook is:
避免从笔记本修改用户shell环境的最佳技术是：

From a terminal:
从终端：
Run standard shell commands but prepend them with uv run
运行标准shell命令，但在它们前面加上uv run
Start uv shell and issue standard shell commands from there
启动uv shell并从那里发出标准shell命令

In a code cell, as long as your notebook is connected to a virtual environment (locally with a tool like uv, or automatically through services like Kaggle and Colab):
在代码单元格中，只要你的笔记本连接到虚拟环境（使用uv等工具本地连接，或通过Kaggle和Colab等服务自动连接）：

Find native Python commands that accomplish your goal
查找完成你的目标的本地Python命令
Use magic commands (starting with %) since they are explicitly mediated by the currently active kernel
使用魔法命令（以%开头），因为它们由当前活动的内核明确调解

NOTE: This works as long as the magic command is not designed to kick off an external python script containing less safe commands.
注意：只要魔法命令不是为了启动包含不太安全命令的外部python脚本而设计的，这就可以工作。

Use shell commands (starting with !) that are explicitly sent to your virtual environment manager
使用明确发送到虚拟环境管理器的shell命令（以!开头）

This is any shell command starting with !uv, !pip, or !python, since those executables will have been provided by the virtual environment.
这是任何以!uv、!pip或!python开头的shell命令，因为这些可执行文件将由虚拟环境提供。
